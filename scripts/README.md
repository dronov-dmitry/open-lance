# Скрипты проекта Open-Lance

В этой директории находятся служебные скрипты для управления базой данных, локальной разработкой и деплоем приложения. Большая часть скриптов представлена в двух вариантах: для Windows (`.ps1`, `.bat`) и для Linux/macOS (`.sh`).

## Управление базой данных (MongoDB)

* **`setup-mongodb.js`**
  Автоматизирует первоначальную настройку базы данных на MongoDB Atlas. Скрипт подключается к кластеру, создает необходимые коллекции (`users`, `tasks`, `applications`), настраивает индексы (включая текстовые) и предлагает добавить тестовые данные для проверки работоспособности системы.

* **`clean-db.ps1`**
  Очищает данные из всех основных коллекций (`users`, `tasks`, `applications`, `reviews`), но при этом сохраняет саму структуру базы данных и все индексы. Удобно использовать для сброса состояния при тестировании. Подхватывает учетные данные из файла `.dev.vars` или переменных окружения. Скрипт написан на PowerShell и запускает Node.js под капотом.

## Локальная разработка

* **`start_dev.bat` / `start_dev.ps1`**
  Скрипты для удобного запуска окружения разработчика. Они одновременно поднимают локальный сервер для фронтенда (на порту `8080`) и локальный сервер Cloudflare Workers для бэкенда (на порту `8787`). Автоматически определяют, нужно ли запускать локальный бэкенд, основываясь на настройках в файле `.env`.

## Развертывание (Деплой) и очистка

* **`deploy.ps1` / `deploy.sh`**
  Основные скрипты деплоя проекта в облако.
  * *Версия для Windows (`deploy.ps1`)* автоматизирует публикацию бэкенда в **Cloudflare Workers** (версия архитектуры v3.0_). Сохраняет необходимые секреты (MongoDB URI, JWT) и настраивает конфигурации фронтенда для подключения к облачному воркеру.
  * *Версия для Linux (`deploy.sh`)* в текущей редакции ссылается на развертывание в AWS Lambda (версия v2.0).

* **`cleanup.ps1` / `cleanup.sh`**
  Удаляют облачную инфраструктуру, созданную при деплое (в первую очередь AWS Lambda, API Gateway и Terraform-ресурсы, если они использовались в более старых версиях). **Внимание:** эти скрипты не удаляют данные из кластера MongoDB Atlas.

* **`test-deploy.ps1`**
  Скрипт проверки готовности к деплою. Он валидирует наличие всех необходимых инструментов в системе: Node.js, npm, AWS CLI, Serverless Framework, Terraform и проверяет конфигурацию аккаунтов. Помогает убедиться, что окружение настроено правильно до начала деплоя.
  
* **`check-region-consistency.ps1` / `check-region-consistency.sh`**
  Проверяют консистентность указанных регионов AWS (US, EU и т.д.) во всех конфигурационных файлах (CLI, Terraform, Backend, Frontend), чтобы избежать ошибок маршрутизации и доступа к ресурсам в старых версиях проекта на базе AWS.
