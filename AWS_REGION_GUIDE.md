# AWS Region Quick Reference Guide

## Быстрый старт: Выбор региона за 1 минуту

**Для пользователей из:**
- 🇺🇸 **США** → `us-east-1` (N. Virginia)
- 🇪🇺 **Европы** → `eu-west-1` (Ireland) или `eu-central-1` (Frankfurt)
- 🇷🇺 **России/СНГ** → `eu-central-1` (Frankfurt) или `eu-west-1` (Ireland)
- 🇨🇳 **Азии** → `ap-southeast-1` (Singapore) или `ap-northeast-1` (Tokyo)
- 🇧🇷 **Южной Америки** → `sa-east-1` (São Paulo)

## Критерии выбора региона

### 1. 📍 Географическая близость

**Правило:** Выбирайте регион ближайший к вашим пользователям

**Почему важно:**
- Меньше задержка (latency)
- Быстрее загрузка страниц
- Лучше пользовательский опыт

**Пример:** 
- Ваши пользователи в Москве → `eu-central-1` (Frankfurt, ~50ms)
- Ваши пользователи в Нью-Йорке → `us-east-1` (N. Virginia, ~5ms)

### 2. 💰 Стоимость

**Самые дешевые регионы:**
1. `us-east-1` (N. Virginia) - самый дешевый
2. `us-west-2` (Oregon)
3. `eu-west-1` (Ireland)

**Более дорогие регионы:**
- `ap-northeast-1` (Tokyo) - примерно на 10-15% дороже
- `sa-east-1` (São Paulo) - примерно на 20-30% дороже
- `af-south-1` (Cape Town) - примерно на 30% дороже

### 3. 🛡️ Соответствие требованиям (Compliance)

**GDPR (Европа):**
- Используйте европейские регионы: `eu-west-1`, `eu-central-1`

**Данные должны храниться в России (152-ФЗ):**
- AWS не имеет регионов в России
- Рассмотрите использование Yandex Cloud или VK Cloud

**HIPAA (медицинские данные, США):**
- Все регионы поддерживают, но нужен Business Associate Agreement (BAA)

### 4. 🚀 Доступность сервисов

**Все сервисы доступны:**
- Lambda, DynamoDB, API Gateway, Cognito работают во всех основных регионах

**Новые сервисы первыми появляются:**
- `us-east-1` получает новые сервисы первым
- Другие регионы получают их с задержкой 3-12 месяцев

## Как проверить задержку до региона

### Способ 1: AWS CloudPing

Онлайн инструмент для проверки задержки:
- https://www.cloudping.info/
- Показывает latency до всех регионов AWS
- Работает прямо в браузере

### Способ 2: Terminal ping

```bash
# N. Virginia (us-east-1)
ping dynamodb.us-east-1.amazonaws.com

# Frankfurt (eu-central-1)
ping dynamodb.eu-central-1.amazonaws.com

# Ireland (eu-west-1)
ping dynamodb.eu-west-1.amazonaws.com

# Singapore (ap-southeast-1)
ping dynamodb.ap-southeast-1.amazonaws.com
```

### Способ 3: AWS CLI

```bash
# Установить aws-ping (если есть)
npm install -g aws-ping

# Проверить все регионы
aws-ping
```

## Как задать регион в конфигурации

### 1. AWS CLI

```bash
aws configure --profile primary
# AWS Access Key ID: AKIA...
# AWS Secret Access Key: wJal...
# Default region name: us-east-1  ← ВАШ РЕГИОН
# Default output format: json
```

### 2. Terraform (terraform.tfvars)

```hcl
aws_region = "us-east-1"  # Замените на ваш регион
```

### 3. Backend Environment (.env)

```bash
AWS_REGION=us-east-1  # Замените на ваш регион
```

### 4. Frontend Configuration (js/config.js)

```javascript
const CONFIG = {
    API: {
        production: {
            baseURL: 'https://xxx.execute-api.us-east-1.amazonaws.com/prod',
            region: 'us-east-1'  // Замените на ваш регион
        }
    }
};
```

## Важные правила

### ✅ МОЖНО

- Использовать один регион для всех ресурсов (рекомендуется)
- Тестировать в одном регионе, production в другом
- Менять регион между деплоями (через `terraform destroy` + `terraform apply`)

### ❌ НЕЛЬЗЯ

- ❌ Использовать разные регионы для Lambda и DynamoDB в одном аккаунте
- ❌ Переносить ресурсы между регионами напрямую (нужен redeploy)
- ❌ Ожидать автоматической репликации данных между регионами

## Частые вопросы

**Q: Можно ли потом сменить регион?**  
A: Да, но нужно будет:
1. Сделать backup данных
2. Запустить `terraform destroy`
3. Изменить регион во всех конфигах
4. Запустить `terraform apply`
5. Восстановить данные

**Q: Какой регион выбрать для России?**  
A: `eu-central-1` (Frankfurt) - минимальная задержка, или `eu-west-1` (Ireland) - чуть дешевле.

**Q: Нужен ли отдельный регион для тестирования?**  
A: Нет, можно использовать один регион, но разные stage (dev/prod).

**Q: Влияет ли регион на Free Tier?**  
A: Нет, Free Tier действует во всех регионах одинаково.

## Быстрая справка: команды проверки региона

```bash
# Узнать текущий регион AWS CLI
aws configure get region

# Посмотреть все профили и их регионы
cat ~/.aws/config

# Проверить, в каком регионе запущены ресурсы
aws ec2 describe-regions --output table

# Узнать список всех доступных регионов
aws ec2 describe-regions --all-regions \
  --query "Regions[].{Name:RegionName}" \
  --output table
```
